# ENStop Project Architecture

## High-Level Structural Overview
- **`supabase/`**: Core backend configuration, database schemas, migration files (`migrations/`), seed files (`seed.sql`), and PostGIS spatial functions.
- **`ui/`**: Single-page React application built with Vite and Tailwind v4.x.
  - **`ui/src/api/`**: Remote API calls and TanStack Query hooks wrapper around the Supabase JS client.
  - **`ui/src/components/`**: Leaflet overlays, map legends, contexts, routing displays, and modular UI presentation layers.
  - **`ui/src/hooks/`**: Context synchronization, geo-snapping utility hooks, and external route queries (e.g., OSRM pathing).
  - **`ui/src/pages/`**: Primary page screens (Map navigation, route manager list, editor canvas, about page).
  - **`ui/src/store/`**: Global Zustand UI states including map viewpoint control and routing coordinates.
  - **`ui/src/styles/`**: Global CSS styling containing Tailwind `@theme` overrides, custom utilities, and third-party styling overrides.
  - **`ui/src/types/`**: Project-wide type models and autogenerated database type definitions.

## Tech Stack & Global State
- **Frontend Core**: React 19 + TypeScript + Vite.
- **Styling**: Tailwind CSS v4.x (CSS-first configuration via `@theme` in `ui/src/styles/index.css`) + Leaflet CSS.
- **Database & Auth**: Supabase client (`@supabase/supabase-js`) accessing Postgres/PostGIS backend databases.
- **Remote Data Queries**: TanStack React Query (`@tanstack/react-query`) managing server-side query states and caching.
- **Global Client State**: Zustand (`zustand`) stores for light, reactive client states:
  - `useMapStore`: Viewport centering, zoom level, selections (stops, routes), user geolocations, and visible route filters.
  - `useRoutingStore`: Navigation directions, origin/destination pins, OSRM routes, map click interaction modes.
- **App Routing**: React Router DOM (`react-router-dom`) handling route detail tabs and deep link mapping.

## Data Flow
1. **Database Queries**: Presentation components trigger TanStack query hooks, fetching spatial data from Supabase client using auto-generated database schemas.
2. **Spatial Logic Execution**: Geospatial operations (e.g. searching nearby stops or querying routes passing through a stop) run via Supabase RPC database functions (`nearby_stops`, `routes_for_stop`) executing PostGIS geographic calculations.
3. **Store ↔ URL Sync**: The `useUrlStoreSync` hook monitors `useRoutingStore` parameters and writes coordinates (`?from=lat,lng&to=lat,lng`) to URL query params, and reads URL state on load or popstate events to keep the store in sync.
4. **Map Render Loop**: Leaflet renders overlays by combining base OpenStreetMap tiles with database-sourced routes and stops, referencing Zustand filters for visibility states and selection highlights.
